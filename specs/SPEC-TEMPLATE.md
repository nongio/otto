## Behavior

- Inputs (user actions, system events, config values)
- Outputs (visual changes, state changes, Wayland events emitted)
- Error/edge cases (what happens when preconditions aren't met)

Be concrete. Prefer "when X happens, Y must occur" over vague descriptions.  
Do **not** reference current code, modules, or data structures — describe the contract, not the implementation.
